Производственных линии
Вся структура цеха занесена в базу данных: бренды → станции → пистолеты → точки сварки → уставки техпроцесса. Это фундаментальная работа, которая позволяет строить любой аналитический и операционный функционал.
Технология

Как работает система

🌐

Веб-приложение

Открывается в любом браузере. Не требует установки. Работает на ПК, телефоне и планшете в локальной сети цеха.

🗄️

База данных

Вся информация хранится централизованно. Любой сотрудник видит актуальные данные в реальном времени.

📱

Мобильная версия

Страницы адаптированы под телефон. Можно регистрировать ТО и дефекты прямо у станка, без ПК.

🔒

Аудит изменений

Каждое изменение уставок записывается с датой и комментарием. История не удаляется.

⚡

Валидация данных

Система проверяет замеры ТО: допуск ±100 А по току и ±50 daN по давлению. Выход за пределы — предупреждение.

🌙

Тёмная тема

Комфортная работа в любых условиях освещения цеха. Переключение сохраняется между сессиями.

ИТО · Уставки техпроцесса

Управление параметрами сварки

  • Уставки хранятся в БД для каждого пистолета и каждой точки
  • Все параметры: давление (daN), коэффициент трансформации, ток 1/2, времена циклограммы
  • Изменение уставок — с обязательным комментарием «зачем меняем»
  • Полный архив: когда, какие параметры, какой комментарий
  • Старые уставки не удаляются — сохраняется история изменений
  • Привязка замеров ТО к конкретной уставке для сравнения факт/план
📊

Что видно в архиве уставок

  • Дата введения уставки
  • Режим сварки
  • Давление, К.Трансформации
  • Ток 1 и Ток 2
  • Комментарий к изменению ← новое
  • Активна / в архиве

Отдел ОТК

Управление дефектами

Регистрация, расследование и закрытие дефектов сварки

ОТК · Дефекты

Регистрация и закрытие дефектов

  • 16 кодов дефектов из техпроцесса (CR, SN, LP, BN, SW и др.)
  • Регистрация: модель авто → номер точки → код дефекта
  • В карточке видно: номер и тип пистолета, бренд, станция — сразу можно найти
  • Workflow простой: Зарегистрирован → Закрыть (без лишних шагов)
  • При закрытии: первопричина + контрмера + ответственный сотрудник
  • История закрытых дефектов с поиском
16 кодов дефектов
CR SN LP BN SW BT MS ME IE MN NA EO BE P MI EMU
Словарь кодов хранится в БД и может пополняться
ОТК · Аналитика

Аналитика дефектов

  • Фильтры: этот месяц / 3 мес / 6 мес / год / свой период
  • Фильтр по модели автомобиля и конкретной станции
  • Диаграмма-пончик: наглядно видно какие коды доминируют
  • Таблица разбивки по станциям — где больше всего проблем
  • Самый частый код дефекта за период — на главном экране
Аналитика отвечает на вопрос: где и что чаще всего ломается?
Позволяет принимать системные решения, а не тушить пожары
